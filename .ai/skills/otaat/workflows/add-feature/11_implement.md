# Step 11 — Implement the Feature

Read `.otat/decision.md`, `.otat/alleyoop.md`, and the contract test from step 02.

Build the feature.

## What to do

1. **Follow alleyoop.md** as your implementation guide. It has the exact files, patterns, and order of operations.

2. **Make the contract test pass.** This is the primary success criterion. If the contract test passes, the core feature works.

3. **Handle edge cases** listed in alleyoop.md. Write additional tests as specified.

4. **Run the full test suite.** All existing tests must still pass. If any break, fix them — but only if the breakage is expected due to the new feature. Unexpected breakage means something is wrong.

## Rules

- Follow the patterns from patterns.md. Do not introduce new patterns.
- Follow the order of operations from alleyoop.md.
- If you hit something alleyoop.md didn't anticipate, handle it minimally and note it for the critique step.
- Do not gold-plate. Implement exactly what was decided, nothing more.
- Commit working code. If the test suite has failures, do not proceed — fix them.

## Output

Working implementation with passing test suite.
