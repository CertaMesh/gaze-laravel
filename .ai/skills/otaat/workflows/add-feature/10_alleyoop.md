# Step 10 — Alley-Oop (Implementation Prep)

Read `.otat/decision.md`, `.otat/design.md`, `.otat/patterns.md`, and the codebase.

Gather everything needed for implementation into a single document. This is the "pass" that makes the "dunk" (step 11) easy.

## What to do

1. **Exact files to create** — Full paths, what each file contains, which pattern it follows from patterns.md.

2. **Exact files to modify** — Full paths, what changes in each file, where in the file the change goes.

3. **Patterns to follow** — For each file, reference the specific existing file it should mirror. Include the pattern name from patterns.md.

4. **Edge cases** — List from acceptance.md, with a note on how each is handled in the implementation.

5. **Test cases beyond the contract test** — Unit tests, additional integration tests, or manual verification steps needed to fully cover the feature.

6. **Order of operations** — The sequence in which files should be created/modified. Dependencies first.

## Rules

- Every instruction must be concrete enough to implement without further decisions.
- Reference specific files, line numbers, function names from the codebase.
- If the decision from step 09 left any ambiguity, resolve it here with evidence from the codebase.
- Do NOT implement. Only prepare.

## Output

Write the result to `.otat/alleyoop.md`.
